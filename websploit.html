Module 1 Information:

You will learn how to find the following vulnerabilities:

1. Injection Flaws: Injection vulnerabilities occur when an application sends untrusted data to an interpreter. Examples include SQL, OS, and LDAP injection. Attackers can exploit these flaws to execute unintended commands or access unauthorized data.

2. Cross-Site Scripting (XSS): This vulnerability allows attackers to inject malicious scripts into web pages viewed by other users. These scripts can bypass access controls and perform actions as if they were the user themselves.

3. Cross-Site Request Forgery (CSRF): CSRF attacks force a logged-on victim's browser to send a forged HTTP request, including the victim's session cookie and any other automatically included authentication information, to a vulnerable web application. 

4. Insecure Direct Object References (IDOR): These vulnerabilities occur when an application exposes a reference to an internal implementation object. Attackers can manipulate these references to gain unauthorized access to data.

5. Security Misconfiguration: This occurs when an application or its environment isn't secured properly. This can lead to unauthorized access to sensitive information or even system data.

6. Insecure Deserialization: Flaws in the deserialization process can allow attackers to execute malicious commands in the application context.

7. Server-Side Request Forgery (SSRF): This vulnerability allows an attacker to make requests to resources internal to the application or network, potentially exposing sensitive data or services.

8. Broken Authentication and Session Management: When authentication or session management functions are improperly implemented, attackers can compromise passwords, keys, or session tokens, or exploit other flaws to assume users' identities.

9. XML External Entity (XXE) Attacks: XXE attacks exploit vulnerabilities in the parsing of XML input. These attacks can lead to the disclosure of internal files, denial of service, or remote code execution.

10. Insufficient Logging & Monitoring: While not a direct attack, the lack of adequate logging and monitoring allows breaches to go undetected and escalate further. Poor incident response can result in the attacker maintaining their presence or causing further damage.

API Vulnerabilities

While APIs (Application Programming Interfaces) share many of the same vulnerabilities as web applications, they also have some unique vulnerabilities due to their nature. Here are some of the most common security vulnerabilities specifically found in APIs:

1. Insecure Endpoint Exposure: When all endpoints of an API are not properly secured, unauthorized users can access these endpoints to fetch, modify, or delete data.

2. Broken Object-Level Authorization (BOLA): This vulnerability happens when an API endpoint exposes objects (like files, database entries, etc.) that a user shouldnâ€™t be able to access. A user might manipulate the ID in the object's URL to gain unauthorized access to another object.

3. Security Misconfiguration: This can happen in APIs when HTTP methods (like PUT, DELETE) are incorrectly configured, CORS headers are misconfigured, or default API keys are used.

4. Broken User Authentication: This is when APIs can't reliably verify the identity of the requester. It could allow unauthorized users to perform actions on behalf of legitimate users.

5. Lack of Rate Limiting: Without proper rate limiting, an API can be subjected to denial-of-service attacks, brute force login attempts, or be overloaded with requests leading to service degradation.

6. Mass Assignment: APIs that use a client-supplied data model for create/update operations can be vulnerable to this. Attackers may guess object properties not intended for modification and send them in a request.

7. Improper Data Filtering: When APIs allow users to fetch data with filters but are not properly implemented, attackers can extract sensitive information.

8. Insufficient Logging & Monitoring: APIs should have detailed logs and real-time monitoring to identify suspicious activities. Without these, a breach could go unnoticed for a long time.

9. Injection: Similar to web applications, APIs are susceptible to different types of injection attacks, including SQL, NoSQL, and OS command injections.

10. Exposed Sensitive Data: APIs that do not properly sanitize responses can leak sensitive data like personally identifiable information (PII), confidential system details, or cryptographic secrets.
